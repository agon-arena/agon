# YouTube Source Preview Contract

This document describes the current runtime contract for YouTube source videos. It is documentation only.

## Scope

This contract covers YouTube videos used as source media in:

- index debate cards, including cards inside swipeable media
- debate page source preview, rendered inside `#debate-source-yt-container`

It does not cover YouTube embeds inside idea/source cards.

## Shared Runtime Shape

Index and debate source previews reuse the same shell generated by `buildIndexYouTubeEmbedHtml(sourceUrl, debateId)`.

Required shell attributes/classes:

- `[data-index-youtube-shell]`
- `[data-index-youtube-poster]`
- `[data-index-youtube-overlay]`
- `.debate-card-youtube-iframe`
- `[data-index-youtube-sound-btn]`
- `.debate-card-youtube-source-sound-toggle`

The debate page wraps that shell in:

- `#debate-source-yt-container`

## Sound Button Contract

The custom sound button is part of the YouTube shell, not an external page control.

Expected placement:

- mobile only visible behavior
- inside `.debate-card-youtube-shell`
- bottom-right of the video rectangle
- `right: 10px`
- `bottom: 10px`
- `z-index: 30`

The current CSS anchor is:

```css
@media (max-width: 768px) {
  .debate-card-youtube-shell > .debate-card-youtube-source-sound-toggle { ... }
}
```

This is intentionally scoped to direct children of the YouTube shell so the button stays inside the video rectangle.

## State Contract

The shell uses dataset flags:

- `data-active`: whether the iframe is active
- `data-user-activated`: whether sound was enabled by user action
- `data-sound-enabled`: UI sound state
- `data-user-started`: whether playback was user-started
- `data-in-view`: index observer visibility state

Sound control functions:

- `bindIndexYouTubeSoundButton(shell, state)`
- `toggleSoundOnIndexYouTubeShell(shell)`
- `enableSoundOnIndexYouTubeShell(shell)`
- `disableSoundOnIndexYouTubeShell(shell)`
- `updateYtSourceSoundButton(button, isSoundEnabled)`

The iframe must include `enablejsapi=1` in its URL so mute/unmute commands can be sent with `postMessageToIndexYouTubeIframe`.

## Debate Page Contract

On the debate page:

- `renderDebateSourcePreview(sourceUrl, preview)` creates `#debate-source-yt-container`
- that container receives `buildIndexYouTubeEmbedHtml(sourceUrl, "")`
- `initDebateYouTubeShell(ytContainer)` binds poster, overlay, and sound behavior
- `cleanupDebateYouTubeShells()` removes the container and blanks the iframe

## Do Not Change Casually

Avoid changing these without a targeted manual mobile check:

- moving the sound button outside `.debate-card-youtube-shell`
- changing `position:absolute`, `right`, `bottom`, or `z-index` for `.debate-card-youtube-source-sound-toggle`
- removing `enablejsapi=1`
- changing the direct-child CSS selector
- changing `initDebateYouTubeShell` to bypass `bindIndexYouTubeSoundButton`
- changing swipe/hotspot behavior around source media

## Suggested Verification

Before and after any runtime change near this area:

- `npm run verify:safe`
- `npm run audit:scale`
- `npm run docs:cleanup`
- `npm run docs:compare`
- `npm run docs:youtube`

Manual check required for runtime changes:

- mobile index card with YouTube source
- mobile debate page source video
- swipeable source card where YouTube is one media item
- button remains inside the video rectangle, bottom-right
- sound toggles on/off without opening the debate modal or navigating away
